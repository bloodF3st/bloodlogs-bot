# bloodLogs

Telegram-бот для логирования сообщений и таймеров неактивности.

Логирует все сообщения из чатов, где ты присутствуешь. Отправляет логи в указанный канал. Уведомляет, если отслеживаемый пользователь молчит дольше заданного порога.

## Как это работает

- Бот добавляется в группы
- На каждое сообщение проверяет, является ли `ADMIN_ID` членом этого чата
- Если да — логирует сообщение (отправитель, чат, текст/метка медиа, время MSK) в лог-канал
- Все команды принимаются только от `ADMIN_ID`; остальные полностью игнорируются без ответов
- Таймеры неактивности работают в фоне и присылают уведомление, когда отслеживаемый юзер замолчал
- Лог-сообщения батчатся: до 5 штук объединяются в одно Telegram-сообщение с задержкой 3с — меньше flood wait, время в логах точное (берётся из `message.date`)

## Настройка в BotFather

1. Открыть [@BotFather](https://t.me/BotFather) → `/newbot`
2. Задать имя и username
3. Скопировать токен → `BOT_TOKEN` в `.env`
4. `/setprivacy` → выбрать своего бота → **Disable**
   Обязательно, иначе бот не будет видеть сообщения в группах, только команды
5. В BotFather → выбрать бота → **Bot Settings** → **Bot to Bot communication** → **Enable**
   Без этого бот не видит сообщения других ботов в группах
6. `/setjoingroups` → **Enable** (опционально, позволяет добавлять бота по ссылке)

## Настройка

Скопировать `.env.example` в `.env` и заполнить:

```env
BOT_TOKEN=123456:ABC-токен-из-BotFather
ADMIN_ID=твой_числовой_telegram_id
DATABASE_URL=sqlite:data/bloodlogs.db
```

`ADMIN_ID` — числовой Telegram ID. Узнать можно у [@userinfobot](https://t.me/userinfobot).

## Запуск

### Из бинарника (рекомендуется)

Скачать последний бинарник из [Releases](../../releases), положить рядом с `.env`:

```
bloodlogs-bot
.env
```

```sh
./bloodlogs-bot
```

### systemd-сервис

```ini
[Unit]
Description=bloodlogs bot
After=network.target

[Service]
WorkingDirectory=/opt/bloodlogs
ExecStart=/opt/bloodlogs/bloodlogs-bot
EnvironmentFile=/opt/bloodlogs/.env
Restart=on-failure
RestartSec=5
MemoryMax=200M

[Install]
WantedBy=multi-user.target
```

```sh
systemctl enable --now bloodlogs-bot
```

### Из исходников

```sh
cargo build --release
./target/release/bloodlogs-bot
```

Требуется Rust 1.75+.

### После первого запуска

1. Бот напишет тебе в личку `ʙʟᴏᴏᴅʟᴏɢs ᴏɴʟɪɴᴇ`
2. Добавь бота в канал/группу куда должны лететь логи и назначь его **администратором** (нужны права на отправку сообщений)
3. Узнай `chat_id` этого канала (например через [@userinfobot](https://t.me/userinfobot) или из URL)
4. Отправь боту в личку:
   ```
   /bchannel -1001234567890
   ```
5. Бот ответит `ʟᴏɢ ᴄʜᴀɴɴᴇʟ sᴇᴛ` — с этого момента все логи идут туда

Без `/bchannel` логи никуда не отправляются — бот получает сообщения, но молчит.

## Команды

Все команды — только для `ADMIN_ID`.

| Команда | Описание |
|---|---|
| `/bchannel <chat_id>` | Установить канал для отправки логов |
| `/bchannel` | Показать текущий лог-канал |
| `/badd [chat_id]` | Добавить чат в белый список логирования |
| `/bdell [chat_id]` | Удалить чат из белого списка |
| `/btimer <user_id> <chat_id> <время>` | Таймер неактивности для юзера в чате |
| `/btimer del <id>` | Удалить таймер по ID |
| `/btimerdel <id>` | Удалить таймер по ID (алиас) |
| `/btimerclear` | Удалить все таймеры в текущем чате |
| `/balltimer <lookback> <threshold>` | Таймеры для всех активных юзеров этого чата за период `lookback` |
| `/btimers` | Список активных таймеров с пагинацией |
| `/bhelp` | Справка по командам |

Формат времени: `30s`, `5m`, `2h`, `1d`. Комбинации: `1h30m`, `2d12h`.

## Что логируется

- Текстовые сообщения → цитата с текстом
- Медиа с подписью → `[PHOTO/VIDEO/…]` + цитата с подписью
- Медиа без подписи → `[PHOTO]`, `[VIDEO]`, `[GIF]`, `[STICKER]` и т.д.
- Вступление в чат / выход из чата
- Имя + ID отправителя (кликабельно), название + ID чата (ссылка на сообщение), время MSK

Что **не** логируется:
- Чаты не из белого списка (не добавленные через `/badd`)

Время в логах — `message.date` из Telegram API (точное серверное время отправки).

## Таймеры неактивности

Таймер срабатывает, когда отслеживаемый юзер не писал в `chat_id` дольше `threshold`. Срабатывает **один раз** — повторное уведомление приходит только после того, как юзер снова написал и снова замолчал.

```
/btimer 123456789 -1001234567890 2h
```

`/balltimer 55m 1h` — установить таймер 1h для всех, кто писал в этом чате за последние 55 минут. Один таймер на юзера на чат; повторный вызов обновляет порог.

> Бот видит только сообщения, отправленные **после** его добавления в чат. После инвайта нужно подождать хотя бы столько, сколько указан `lookback`, чтобы в `chat_activity` накопились записи — иначе `/balltimer` вернёт «нет активных пользователей».

## Интеграция с blood-harvest

bloodlogs-bot можно использовать как приёмник уведомлений для userbot'а [blood-harvest](https://github.com/bloodF3st/blood-harvest-src).

Добавить в `.env` blood-harvest:

```env
BLOODLOGS_TOKEN=<токен этого бота из BotFather>
BLOODLOGS_CHANNEL_ID=<chat_id лог-канала>
```

`BLOODLOGS_CHANNEL_ID` — тот же канал, что задан через `/bchannel`. Узнать текущий:

```bash
sqlite3 bloodlogs.db 'SELECT value FROM bot_settings WHERE key="log_channel";'
```

После этого:
- **`.logs`** в blood-harvest — сообщения из отслеживаемых чатов приходят в лог-канал в едином формате с логами bloodlogs-bot
- **`.timer`** в blood-harvest — уведомления о сработавших таймерах приходят тебе в личку от этого бота

Оба бота работают независимо. Если один недоступен — второй продолжает работу.

## Уведомления

При запуске: `ʙʟᴏᴏᴅʟᴏɢs ᴏɴʟɪɴᴇ` (первый запуск) или `ʙʟᴏᴏᴅʟᴏɢs ʀᴇsᴛᴀʀᴛᴇᴅ`.

Если long-poll падает более 5 раз за 60 секунд — бот выходит с кодом 1, systemd перезапускает его.
